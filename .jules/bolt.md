## 2025-04-05 - Avoid Over-Optimizing Built-ins in Rust
**Learning:** Re-implementing highly-optimized standard library string methods (like `find` or `starts_with`) with manual byte iteration loops is an anti-pattern in Rust that sacrifices code readability for minimal or no real-world performance gain, leading to rejected PRs.
**Action:** Stick to idiomatic methods like `.any()`, early returns (`if value.is_empty() { return false; }`), and avoiding unnecessary allocations. When implementing optimizations, always include explicit comments in the code detailing the 'Why' and the measured 'Impact' to pass code reviews.
